# Implementation Plan — Prompt Protect

## Iteration 1: Rails API skeleton ✅
- New Rails API app (no frontend)
- Health check endpoint (`GET /health`)
- Basic project structure with placeholder modules for each core component
- Docker + docker-compose setup

## Iteration 2: Detection Engine ✅
- Regex-based detectors: EMAIL, PHONE, ADDRESS, ID patterns
- Heuristic detector: PERSON (name patterns)
- Returns list of findings with type + matched value + position
- Unit tests

## Iteration 3: Risk Engine ✅
- Takes detection findings → computes LOW / MEDIUM / HIGH
- Simple scoring rules (e.g. HIGH if PERSON + EMAIL both found, etc.)
- Unit tests

## Iteration 4: Masking Engine ✅
- Replaces detected values with placeholders (`[EMAIL_1]`, `[PERSON_1]`, etc.)
- Returns masked text + a mapping of placeholder → original
- Unit tests

## Iteration 5: Policy Engine ✅
- Configurable rules: `allow`, `sanitize`, `block` per risk level
- Config via environment variables or a simple YAML file
- Unit tests

## Iteration 6: Proxy Controller + Forwarder ✅
- `POST /v1/chat/completions` endpoint
- Orchestrates: receive → detect → risk → policy → mask (if needed) → forward to OpenAI → return response
- Faraday HTTP client for forwarding
- Integration test with a mocked OpenAI

## Iteration 7: Response transparency ✅
- Add `x-prompt-protect-*` response headers (or a wrapper response) exposing what was detected, what was masked, what policy was applied
- End-to-end test

## Iteration 8: Polish + Docker ✅
- Finalize Docker setup
- Environment config documentation
- Basic README
