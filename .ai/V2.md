# Implementation Plan — Prompt Protect V2

## Iteration 1: spaCy NER Sidecar

Replace the regex-based `PersonDetector` with a proper NER detector that calls a spaCy sidecar.
Everything else in the pipeline (Risk, Policy, Masking, Proxy) stays unchanged.

### Step 1: spaCy service
- New `services/spacy/` directory
- `main.py` — FastAPI app with two endpoints:
  - `POST /detect` — accepts text, returns entities
  - `GET /health` — liveness check
- `requirements.txt` — fastapi, uvicorn, spacy
- `Dockerfile` — Python slim, installs spaCy + `en_core_web_sm` model

API contract:
```
POST /detect
{ "text": "James Carter called Kat P. from Acme Corp" }

→ { "entities": [
    { "text": "James Carter", "label": "PERSON", "start": 0,  "end": 12 },
    { "text": "Kat P.",       "label": "PERSON", "start": 20, "end": 26 },
    { "text": "Acme Corp",    "label": "ORG",    "start": 32, "end": 41 }
  ]}
```

### Step 2: NerDetector (Rails)
- New `app/services/prompt_protect/detectors/ner_detector.rb`
- Calls spaCy service via Faraday
- Maps spaCy labels → our types: `PERSON` → `:person`
- Handles timeouts and connection errors gracefully

### Step 3: Fallback strategy
- If spaCy service is unreachable → silently fall back to existing `PersonDetector`
- Controlled via env var: `SPACY_ENABLED=true/false`
- `SPACY_SERVICE_URL` env var (default: `http://spacy:5001`)
- Keeps the product working even if the sidecar is down

### Step 4: DetectionEngine update
- Swap `PersonDetector` for `NerDetector` in the `DETECTORS` list
- No other pipeline changes

### Step 5: docker-compose update
- Add `spacy` service — internal only, not exposed externally
- Still just `docker compose up`

### Step 6: Tests
- Unit tests for `NerDetector` with mocked HTTP (WebMock)
- Fallback test — spaCy unreachable → PersonDetector kicks in
- Integration test against vehicle loan + name samples

### What stays the same
- All existing regex detectors (EMAIL, PHONE, ADDRESS, ID) — untouched
- `PersonDetector` — kept as fallback
- Risk, Policy, Masking, Forwarder, Proxy — zero changes
- `docker compose up` — still one command

### Out of scope for V2 Iteration 1
- ORG / LOCATION as new entity types (Iteration 2)
- gRPC transport (Iteration 3)
- Multilingual models
- Fine-tuning
